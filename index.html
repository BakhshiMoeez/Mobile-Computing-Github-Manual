Github Manual

Introduction to Github

GitHub is a web-based interface that uses Git, the open source version control software that lets multiple people make separate changes to web pages at the same time. As Carpenter notes, because it allows for real-time collaboration, GitHub encourages teams to work together to build and edit their site content.

Purpose & Importance of using Github

It's used for storing, tracking, and collaborating on software projects. It makes it easy for developers to share code files and collaborate with fellow developers on open-source projects. GitHub also serves as a social networking site where developers can openly network, collaborate, and pitch their work

Steps to use the Git and Github repositories

  • Initialization of Git and repository
  • Setting up origin and git status
  • Git Add and commit
  • Git Branch & push
  • Git Clone
  • Git Merge Branches

Initialization of Git and repository

Inorder to make a repository on Github we have to make an account on Github then with our profile pic we have to select arrow button then go to Make new repository, fill all the information and this way repository is created.
After making the folder with the fie to be uploaded open the Git bash and use the command git init.

Setting up origin and git status

The step 2 of git includes setting up the origin where we would have to write the command git remote add origin (repository path)
if we want to look the status of the current directory we may use git status. The files which maybe modified or new shall be stated and if no file is added, deleted or modified then it would tell that it is uptodate with origin/main ie our current branch.

Git add and commit

After the changes are made and git status also shows some modified or newly added files we shall now use git add . and git commit -m "Appropriate message"

Git Branch and Push

Inorder to push the current files into the repository we may use the command git push where on origin main branch the new updated if any files would be uploaded.
We and also create new branches and then add data to those branches by using the command git branch (name of branch) and to go to certain branches we may use the command git checkout (name of branch)

Git Clone command

if we are to write the command git clone (path to the repository) then we maybe able to clone the project in our folder and then again push it but for cloning there might be a need to put some username and password of your account in the git bash terminal.
